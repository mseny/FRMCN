Plan Técnico y Curricular

Análisis estructurado y plan de implementación para un pipeline de inteligencia comercial impulsado por IA.

Lead Fullstack Engineer • MVP Plan

1 Lectura del Proyecto

"No es un dashboard. Es un sistema de razonamiento automatizado sobre datos de negocio. Esa distinción define todo el stack."

El AI Decision Hub se conceptualiza como un pipeline de inteligencia comercial robusto, estructurado en tres capas operativas fundamentales que transforman datos crudos en decisiones accionables:

1. Ingesta de Datos

Recolección de fuentes heterogéneas (SAP, GA4, Zoho, Connectif, Paid Media) en una capa de staging unificada.

2. Procesamiento AI

Motor de razonamiento (Claude/Gemini) que ingiere métricas y las transforma en lenguaje de decisión empresarial.

3. Output Estructurado

Entrega de la información en forma de reportes, alertas tempranas y recomendaciones accionables para la dirección.

Arquitectura

Stack Tecnológico Recomendado

Selección de tecnologías robustas orientadas a la creación de un MVP productivo y escalable.

Backend & Orquestación

  • Python 3.12: Lenguaje principal del pipeline.
  • FastAPI: API REST para conectar fuentes y servir outputs.
  • LangChain / LlamaIndex: Orquestación de llamadas a AI con contexto.
  • APScheduler: Ejecución del pipeline semanal automatizado.
  • Pydantic & Pandas: Validación de schemas y cálculo de desviaciones (WoW/MoM).

Integraciones de Datos

  • Google Sheets API: Capa de staging universal. (Ideal para MVP: export SAP CSV → Sheets).
  • GA4 Data API: Tráfico y conversion rate.
  • Zoho CRM API: Segmentos RFM, BBDD clientes.
  • Connectif API: Métricas de email marketing.

Frontend & Output

Next.js 14 (App Router) Tailwind CSS shadcn/ui Recharts React Query

Interfaz web del hub para visualizar KPIs, sincronizar estado servidor/cliente y aprobar decisiones.

DB & Infraestructura

  • PostgreSQL (Supabase): Persistencia de reportes y alertas.
  • Redis: Caché de resultados AI (ahorro de costes).
  • Google Cloud Run & Vercel: Deploy serverless (Backend/Frontend).
  • Cloud Scheduler & Actions: CI/CD y triggers semanales.

Currículo Propuesto

Plan de Formación: 5 Clases

Al final del programa, el alumno dispondrá de un MVP completamente funcional del AI Decision Hub.

Clase 1: Fundamentos — Claude API como Motor de Decisión

Concepto Central: Claude no es un chatbot, es un motor de razonamiento que consume datos estructurados y produce decisiones.

Qué aprende el alumno:

  • Diferencia entre prompt naïve vs. prompt de sistema estructurado.
  • Arquitectura de llamada a Claude API con rol analítico.
  • Pasar KPIs como contexto estructurado (JSON) y recibir Insight/Diagnóstico.
  • Uso de Pydantic schemas para validar respuestas de la IA.
Práctica

Construir función Python que reciba un dict con KPIs (revenue, traffic, cr) y devuelva el insight estructurado.

Clase 2: Ingesta de Datos Reales — Sheets + GA4 API

Concepto Central: El pipeline empieza con datos reales, no mocks. Conectar las fuentes del MVP.

Qué aprende el alumno:

  • Autenticación con Google Cloud (Service Account, OAuth2).
  • Sheets API (como hub de datos para el equipo no-técnico) y GA4 Data API.
  • Construcción de una capa DataCollector en Python.
  • Cálculo automático de desviaciones (actual - target).
Práctica

Pipeline que extrae datos de GA4 + Sheets (revenue SAP) y produce un JSON consolidado listo para procesar.

Clase 3: Gemini + Arquitectura Multi-AI

Concepto Central: Orquestar múltiples IAs según sus fortalezas nativas.

Qué aprende el alumno:

  • Gemini API: Procesamiento de contexto amplio y lectura de PDFs largos.
  • LangChain: Orquestar llamadas secuenciales (Gemini extrae → Claude razona).
  • Chain of Thought prompting y manejo de errores (alucinaciones).
  • Introducción a LangSmith para observabilidad del modelo.
Práctica

Pipeline completo: Gemini extrae KPIs de un PDF → Claude genera diagnóstico → Pydantic valida.

Clase 4: Backend FastAPI + Scheduler + DB

Concepto Central: Pasar de script Python a sistema productivo con API, persistencia e ejecución automática.

Qué aprende el alumno:

  • Estructurar FastAPI con endpoints: /run-pipeline, /reports.
  • APScheduler para ejecución automática (Ej. lunes 6am).
  • Supabase / PostgreSQL: Persistencia de reportes e histórico.
  • Sistemas de alertas automatizadas y Deploy en Cloud Run (Docker).
Práctica

Backend programado que corre el pipeline, guarda en BD y envía email resumen a los directivos.

Clase 5: Frontend Next.js — El Hub de Decisión

Concepto Central: UI que convierte datos en decisiones visibles para directivos no-técnicos.

Qué aprende el alumno:

  • Arquitectura Next.js 14 App Router.
  • Consumir FastAPI desde el frontend con React Query.
  • Visualización de KPIs con Recharts (target vs actual, semáforos).
  • Sistema "Human-in-the-loop": botón para aprobar recomendaciones AI.
Práctica

Dashboard UI completo desplegado en Vercel con tarjetas estructuradas, insights y acciones aprobables.
