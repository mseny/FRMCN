1 El Modelo Mental: Una Redacción, Varios Cerebros

La limitación más crítica de cualquier sistema conversacional tradicional es la saturación de la ventana de contexto. Cuando forzamos a una sola sesión de chat a leer bases de datos completas, realizar análisis, depurar código y redactar informes de forma secuencial, el modelo sufre de "ruido cognitivo" severo, activa compactaciones de memoria y pierde precisión (Lost in the Middle).

Patrón Orchestrator-Workers

El orquestador principal (Claude Opus o Sonnet) actúa como un director ejecutivo. Tiene el objetivo global en mente, planifica, divide tareas y delega la ejecución de micro-tareas complejas a subagentes especialistas, recibiendo únicamente un resumen con el entregable consolidado.

Aislamiento de Contexto

Cada subagente corre en su propia ventana de contexto aislada. Esto garantiza que sus micro-decisiones no saturen la conversación principal y que las herramientas de ejecución (como Bash o Python) se acoten estrictamente a lo que requiere su especialidad.

Símil de la Redacción de Noticias: En una redacción de prensa, el redactor jefe no escribe la sección de deportes, economía y clima de su puño y letra al mismo tiempo. Delega en tres redactores especializados. Cada especialista trabaja de manera independiente y le entrega una columna limpia y sintetizada. El redactor jefe solo ensambla la portada final.

2 La Tríada de Claude: Elegir el Entorno Adecuado

Anthropic ofrece tres entornos diferenciados según el grado de control técnico y la naturaleza del flujo de trabajo de cada usuario.

Claude Chat (web/móvil)

La interfaz conversacional tradicional.

  • Preguntas y respuestas rápidas.
  • Redacción creativa y formateo ágil.
  • Sin acceso a archivos locales o herramientas de sistema.
Perfil: General
Productividad

Claude Cowork

El agente de escritorio de conocimiento.

  • Ejecución de flujos complejos multi-paso.
  • Lectura y escritura de carpetas locales.
  • Integración visual de plugins y conectores.
Perfil: Knowledge Worker / Manager

Claude Code (CLI)

La interfaz de comandos agéntica.

  • Operaciones autónomas sobre repositorios.
  • Ejecución directa de comandos bash y scripts.
  • Control granular de agentes y skills en .md.
Perfil: Desarrollador / Analista Técnico

3 La Anatomía Extensible: Skills, Conectores y Plugins

Para automatizar tu entorno de forma robusta en Cowork o Code, debes comprender cómo se expanden las capacidades del modelo.

Skills

Son el cómo hacer las cosas. Archivos de Markdown que especifican procedimientos, reglas de formato e instrucciones precisas de ejecución que Claude carga bajo demanda.

Carga Bajo Demanda

Connectors (MCP)

Son las ventanas al mundo real. Basados en el estándar de Anthropic (Model Context Protocol), enlazan de manera nativa a Claude con apps como Google Drive, Slack, Notion o HubSpot.

Integración de Datos

Plugins (.plugin)

Son el paquete empaquetado total. Un archivo ZIP o carpeta que reúne en una sola unidad Skills, subagentes (Agents), conectores MCP y Hooks de sistema automatizados.

Ecosistema Completo

El Secreto de la Progressive Disclosure (Carga Escalonada)

En Claude Code y plugins estructurados, los skills se diseñan con carga escalonada de 3 capas. Esto reduce los costes en tokens radicalmente en un 90%:

Capa 1: Metadata (Fijo)

Claude mantiene fijos en la ventana de contexto solo el name y description de todos los skills del directorio. Esto permite al orquestador saber que existen gastando muy pocos tokens fijos.

Capa 2: Body (Al Activarse)

Solo cuando Claude decide que un skill se ajusta a la petición, el cuerpo del fichero SKILL.md con todas sus instrucciones detalladas se vuelca e inyecta en el contexto de la sesión.

Capa 3: References (Bajo Demanda)

La documentación extendida de la carpeta references/ nunca entra automáticamente al contexto. Claude debe llamar de manera explícita a la herramienta de lectura para leerlos.

4 Cómo Crear Agentes y Subagentes

Dependiendo de si utilizas la terminal con Claude Code o la interfaz de escritorio con Claude Cowork, la creación y configuración de agentes se adapta con flujos optimizados. A continuación, exploramos los dos caminos sin desbordar el contexto:

En Claude Code (CLI)

Anthropic recomienda encarecidamente utilizar el asistente interactivo integrado. Esto evita errores de sintaxis en el formato YAML del frontmatter y agiliza el proceso de desarrollo.

$ claude /agents

Este comando despliega una interfaz paso a paso que te guía para definir el nombre, la descripción (triggers), el modelo y las herramientas permitidas.

Ruta de guardado automático: .claude/agents/ (A nivel de proyecto)

En Claude Cowork (Desktop)

En el ecosistema visual de escritorio, los subagentes se integran mediante empaquetados modulares o asistentes conversacionales integrados:

  • Plugin Create Asistente visual integrado que te guía en lenguaje natural para programar y consolidar tus subagentes desde un chat.
  • Archivos .plugin (ZIP) Puedes empaquetar tus archivos Markdown y subirlos como un archivo comprimido que se despliega al instante en tu Workspace.
Estructura del empaquetado: plugin/agents/mi-agente.md
