1 AntiGravity: Programar con IA en el IDE

Un IDE (Integrated Development Environment) es el software donde los desarrolladores escriben código (ej. VS Code, Cursor). AntiGravity representa la nueva generación de IDEs con IA integrada nativamente, permitiendo autocompletar código, chatear con el proyecto y generar funciones enteras desde lenguaje natural.

¿Para qué sirve en este curso?

Para construir el AI Decision Hub: scripts de conexión a APIs, procesamiento de datos locales y generación de reportes automáticos sin necesidad de ser un programador experto.

Analogía: Si el código es una receta de cocina, el IDE es la cocina completa. AntiGravity añade un chef asistente que te sugiere los ingredientes, te avisa si vas a quemar algo y puede cocinar partes enteras del plato por ti.

Autocompletado

La IA predice y sugiere la siguiente línea o bloque entero mientras escribes, basándose en el contexto del archivo. (Como el autocorrector del móvil, pero súper inteligente).

Chat con Codebase

Permite hacer preguntas al IDE: "¿Qué hace esta función?" o "Refactoriza esto para que sea limpio", dándole acceso de lectura a todo tu proyecto.

Inline Editing

Seleccionas un bloque de código y le pides a la IA que lo modifique directamente allí mismo, sin tener que salir a una ventana de chat externa.

Contexto Total

El IDE con IA "lee" todos los archivos de tu proyecto simultáneamente para dar sugerencias que sean coherentes con la arquitectura y variables que ya existen.

2 APIs y CLI: Hablar con las Herramientas

API

Application Programming Interface. Contrato que permite a dos programas comunicarse.

Símil: El menú del restaurante. Pides algo (al camarero/API) y la cocina (sistema) te trae el plato.

REST

Estilo de comunicación para APIs web. Usa peticiones HTTP (GET, POST, PUT, DELETE).

El estándar de la industria para enviar y recibir datos en la web moderna.

JSON

Formato ligero de texto para estructurar datos (JavaScript Object Notation).

Símil: Un formulario bien rellenado. Campos claros y valores definidos, legible por máquinas.

CLI

Command Line Interface. Interfaz donde ejecutas comandos escribiendo texto.

Símil: La interfaz visual es un coche automático; la CLI es uno manual. Más control y velocidad.

Integrando la API de Anthropic

API Key: Credencial única que identifica quién hace cada petición. Es como tu DNI para el servicio.

Endpoint principal: https://api.anthropic.com/v1/messages

Parámetros: model, max_tokens, messages, system.

1. Llamada por Terminal (cURL)
curl https://api.anthropic.com/v1/messages \ --header "x-api-key: $ANTHROPIC_API_KEY" \ --header "anthropic-version: 2023-06-01" \ --header "content-type: application/json" \ --data '{ "model": "claude-3-5-sonnet-20241022", "max_tokens": 1024, "messages": [ {"role": "user", "content": "Analiza estos KPIs..."} ] }'
2. Llamada con Python (SDK)
import anthropic client = anthropic.Anthropic( api_key="TU_API_KEY", ) message = client.messages.create( model="claude-3-5-sonnet-20241022", max_tokens=1024, system="Eres el AI Decision Hub.", messages=[ {"role": "user", "content": "Calcula el revenue..."} ] ) print(message.content[0].text)
Interoperabilidad

MCP: Conectar el Mundo Real

Model Context Protocol (MCP) es un estándar abierto creado por Anthropic que permite a los modelos de IA conectarse de forma segura y estructurada con herramientas externas, bases de datos, APIs y sistemas de archivos.

Analogía: MCP es como un adaptador universal de enchufes para la IA. Cada herramienta (Slack, Sheets, SAP) tiene su propio "enchufe", y MCP proporciona el adaptador para que Claude pueda conectarse a todas ellas usando el mismo estándar.

¿Por qué MCP y no una API normal?

Las APIs requieren que construyas la integración manualmente paso a paso. MCP define un protocolo estándar para que las herramientas se "expongan" directamente al modelo, permitiéndole llamarlas de forma autónoma cuando las necesite.

El Agente Verdadero

Si Claude sin herramientas es un consultor que solo puede hablar, con MCP es un consultor que además puede abrir tu CRM, leer tus datos en vivo, enviar emails y actualizar tu base de datos de forma autónoma.

MCP Server

Programa que expone capacidades. Ej: Un servidor MCP de Google Sheets permite leer/escribir hojas.

MCP Client

El modelo o agente (ej. Claude) que consume las herramientas expuestas por los servidores.

Tools

Funciones específicas que el servidor permite ejecutar. Ej: read_file, search_slack.

Resources

Datos o contexto que el servidor pone a disposición (archivos locales, URLs, bases de datos).

Laboratorio Agentic

Ejecución secuencial: Del IDE al Agente conectado.

1. Despliegue del IDE

  • Instala y configura AntiGravity.
  • Conéctalo con la carpeta de tu proyecto local.
  • Ejercicio: Pide al Chat del IDE: "Crea un script en Python que lea el archivo data.csv y calcule el revenue total."

2. Conexión API

  • Obtén tu API Key de Anthropic Console.
  • Ejecuta la primera llamada por terminal (cURL) comprobando la respuesta cruda en JSON.
  • Ejercicio: Diseña el script en Python que envíe los datos calculados en el paso 1 a Claude para que redacte un análisis.

3. Autonomía MCP

  • Instala el MCP Server de *filesystem* (para que la IA pueda leer tu disco duro).
  • Configura el archivo MCP en Claude Cowork.
  • Ejercicio: Pide a Claude que lea un Google Sheet y genere el reporte mensual de forma 100% autónoma.
