1 ¿Qué es n8n y por qué usarlo?

n8n (pronunciado "nodemation") es una plataforma de automatización de flujos de trabajo basada en nodos. Al igual que Zapier o Make, permite conectar diferentes aplicaciones (Gmail, Slack, OpenAI, bases de datos) sin necesidad de escribir código complejo desde cero.

Paradigma Visual

Los flujos se construyen uniendo "nodos" con cables. Cada nodo es un paso lógico: uno extrae datos, otro los procesa con JavaScript (si quieres) y el último los envía a otra plataforma.

La Diferencia Técnica

A diferencia de Zapier (enfocado al usuario final), n8n está pensado para perfiles técnicos y operaciones complejas: permite escribir código dentro de los nodos, gestionar bucles complejos y autoalojarse.

Los Bloques de Construcción

Triggers (Gatillos)

Nodos especiales que inician el flujo. Ej: "Cuando llegue un email nuevo" o un Webhook que espera un aviso de otra app.

Actions (Acciones)

Nodos que hacen cosas: crear una fila en Google Sheets, mandar un mensaje a Slack, pedir un texto a OpenAI.

Core/Logic Nodos

Nodos que modifican los datos: IF (condicionales), Switch, Code (escribir JavaScript para filtrar datos).

2 La Lógica de los "Datos Itemizados"

Si no entiendes cómo fluyen los datos en n8n, no podrás construir automatizaciones avanzadas. Todo en n8n es una matriz (Array) de objetos JSON.

En plataformas como Zapier, si pides "Dame los clientes de la base de datos", recibes un bloque gigante de texto. En n8n, la información fluye como una cinta transportadora de cajas independientes.

Ejecución Automática por Item

Si un nodo produce 3 items (ej: 3 correos sin leer), el siguiente nodo al que esté conectado se ejecutará 3 veces de forma automática, una vez por cada item. No tienes que programar un bucle manual (Loop/For-each); n8n lo hace por diseño.

Símil: Piensa en n8n como una fábrica de embotellado. El nodo Trigger vierte 5 botellas vacías en la cinta. El siguiente nodo (Llenar Botella) no actúa sobre el "grupo", actúa individualmente sobre la botella 1, luego la 2, luego la 3...

Estructura JSON de Items en n8n
// Así viajan los datos por el "cable" entre nodos: // Un Array [] que contiene 2 Objetos JSON {} (Items) [ { "json": { "nombre": "Carlos", "email": "carlos@mail.com" } }, { "json": { "nombre": "Elena", "email": "elena@mail.com" } } ] // El siguiente nodo "Enviar Email" se ejecutará // automáticamente 2 veces.

3 Nodos Esenciales y el Arte de "Parsear"

Conocer los nodos principales te permite construir el 80% de las automatizaciones. Pero antes, debes entender cómo traducir la información.

¿Qué es Parsear?

"Parsear" (del inglés Parsing) es el proceso de analizar una cadena de texto sin formato y convertirla en una estructura de datos organizada (como los objetos JSON que acabamos de ver) para que n8n pueda leerla y manipularla.

Si recibes un archivo CSV gigante o la respuesta en texto plano de una IA, n8n solo ve una "caja de letras". Al pasarlo por un nodo que lo parsea, n8n extrae mágicamente las columnas y filas convirtiéndolas en "Items" individuales. Es el traductor universal entre sistemas.

Los Nodos más Comunes

Webhook

El Trigger por excelencia. Crea una URL pública que se queda "escuchando". Cuando otra app envía datos a esa URL, el flujo se dispara instantáneamente.

HTTP Request

La navaja suiza. Sirve para conectarse a CUALQUIER API de internet (GET, POST), incluso si n8n no tiene un nodo oficial para esa aplicación.

Edit Fields (Set)

Mutador de datos. Permite crear, renombrar o borrar campos dentro de un Item. Ideal para limpiar los datos antes de guardarlos en tu base de datos.

Switch / If

Enrutadores lógicos. "Si el email contiene la palabra 'Urgente', envíalo por la ruta A hacia Slack. Si no, envíalo por la ruta B hacia el archivo de Excel".

Code

El nodo sin límites. Escribe JavaScript o Python puro para procesar arrays complejos, parsear datos propietarios o aplicar lógicas matemáticas.

Schedule Trigger

El temporizador automático (Cron job). Inicia un flujo todos los lunes a las 9:00 AM o cada 15 minutos exactos sin intervención humana.

4 Formas de Ejecución (Despliegue)

La mayor ventaja competitiva de n8n es dónde se ejecuta. Puedes delegarlo en sus servidores o tener el control absoluto de tus datos en los tuyos.

n8n Cloud

SaaS Gestionado

La versión tradicional de software como servicio. Pagas una suscripción mensual, te registras en su web y empiezas a automatizar. Ellos mantienen los servidores.

  • Cero mantenimiento.
  • Soporte técnico oficial.
  • Límites de ejecuciones al mes.

n8n Desktop

Local & Gratuito

Una aplicación que descargas (Mac/Windows). Convierte tu ordenador en el servidor. Excelente para aprender y hacer prototipos sin pagar.

  • 100% Gratuito y fácil de instalar.
  • Datos se quedan en tu PC.
  • Si apagas el PC, se apaga la automatización.

Self-Hosted

Máximo Control

Instalas el código de n8n en tu propio servidor (AWS, DigitalOcean) mediante Docker o npm. Es la opción preferida por empresas tecnológicas y bancos.

  • Privacidad total (ideal GDPR/HIPAA).
  • Sin límites de ejecución artificiales.
  • Requiere conocimientos de DevOps.
Modelo de Negocio

5 La Licencia "Fair-Code"

¿Por qué puedo instalar n8n gratis en mi servidor empresarial?

Código Abierto vs. Fair-Code

n8n no es Open Source (Código Abierto) estricto (como Linux o WordPress), aunque su código fuente es visible en GitHub. Utiliza una licencia llamada Fair-Code.

  • Lo que PUEDES hacer gratis: Descargar el código, instalarlo en tu servidor privado y usarlo internamente en tu empresa para automatizar todas las tareas que quieras sin pagar un céntimo en suscripciones.
  • Lo que NO puedes hacer: Coger el código de n8n, meterlo en tu propia web y vender la automatización como un servicio (SaaS) a tus clientes para competir contra ellos, a menos que adquieras una licencia de pago corporativa.
Parte Práctica

Laboratorio de Conexión y Migración

Vamos a construir tu primer flujo leyendo datos de una tabla y enviándolos procesados a otra.

1. El Flujo de Sincronización (Sheets)

Conectaremos una tabla de origen con una tabla de destino para automatizar el trasvase de información corporativa.

  • A
    Leer Datos (Origen): Añade un nodo de Google Sheets y configúralo como "Get Many Rows". Dile que lea los datos de una hoja llamada 'Nuevos Leads'. n8n extraerá cada fila como un Item individual.
  • B
    Limpiar / Mapear: Conecta un nodo Edit Fields (Set). Úsalo para asegurarte de que solo pasas el "Nombre" y el "Email", descartando columnas innecesarias o cambiando el nombre del campo.
  • C
    Escribir Datos (Destino): Conecta otro nodo de Google Sheets. Configúralo como "Append Row" (Añadir fila) apuntando a una hoja llamada 'CRM Central'. Ejecuta el flujo y observa la magia.

2. Transformación Avanzada (Code Node)

A veces los datos no llegan en el formato correcto (ej. necesitas parsear un texto y transformarlo). Usaremos JavaScript básico en el nodo Code.

Code Node (JavaScript)
// Iteramos sobre los datos recibidos de la hoja (Items) for (const item of $input.all()) { // Creamos un "saludo" uniendo "Hola " y la columna "nombre" item.json.saludo_personalizado = "Hola " + item.json.nombre; // Convertimos el email a minúsculas para estandarizar if (item.json.email) { item.json.email = item.json.email.toLowerCase(); } } // Devolvemos el array modificado al siguiente nodo (Destino) return $input.all();
